Migrating from cnetworkmanager to nmcli

This is not meant to be a two-way comparison. I intend cnetworkmanager to rest and nmcli to take its place.
cnnetworkmanager nmcli
general
cnetworkmanager --state nmcli nm
nmcli -t -f state nm
cnetworkmanager --online=BOOL nmcli nm sleep
nmcli nm wakeup
cnetworkmanager --we
cnetworkmanager --wireless-enabled 		nmcli nm wifi
cnetworkmanager --whe
cnetworkmanager --wireless-hardware-enabled 		nmcli -t -f wifi-hardware nm
cnetworkmanager --wifi=BOOL nmcli nm wifi BOOL
devices
cnetworkmanager -d
cnetworkmanager --device-list 		nmcli dev
cnetworkmanager --di DEVICE
cnetworkmanager --device-info DEVICE 		nmcli dev list iface DEVICE
cnetworkmanager -a, -n, --ap-list, --ap, --nets nmcli dev wifi list
connections
cnetworkmanager -u
cnetworkmanager -s
cnetworkmanager --usrcon
cnetworkmanager --syscon 		nmcli con
nmcli con list user
nmcli con list system
cnetworkmanager --activate-connection=[KIND],ID,[DEV],[APMAC] nmcli con up id ID iface DEV ap APMAC
cnetworkmanager -c
cnetworkmanager --actcon 		nmcli con status
cnetworkmanager -C
cnetworkmanager --connect 		N/A, nmcli cannot serve as a connection service,
nor it can add a connection to a service (which cnetworkmanager can't either)
